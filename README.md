# AI Chief 🤖🍳

AI Chief — локальный помощник-повар: вы отправляете фотографию набора ингредиентов, а модель ИИ подсказывает рецепты, подходящие блюда и пошаговые инструкции по приготовлению. Проект ориентирован на удобство использования, приватность (работа локально или на собственном сервере) и расширяемость.

Описание
--------
AI Chief анализирует изображение с продуктами, распознаёт ингредиенты и предлагает рецепты, варианты замены, список покупок и план приготовления. Поддерживает как простой веб-интерфейс, так и REST API для интеграции в другие приложения.

Ключевые возможности
--------------------
- Распознавание ингредиентов на фотографии (Computer Vision).
- Подбор рецептов по набору ингредиентов (с учётом ограничений: аллергии, диет).
- Пошаговые инструкции и список покупок.
- Рекомендации по заменам ингредиентов и подсказки по технике приготовления.
- Локальный режим работы (без отправки фото на внешние сервисы) — если модель запускается локально.
- REST API для автоматизации и интеграции.
- Возможность добавления собственных рецептов/баз знаний.

Технологии
----------
- Python 3.8+
- Рекомендуемые библиотеки (пример): FastAPI, Uvicorn, Pillow, torchvision / TensorFlow / PyTorch, transformers (если используются модели NLP), OpenCV, scikit-image
- Хранилище рецептов: JSON / SQLite / PostgreSQL (опционально)
- Docker для контейнеризации (опционально)

Требования
---------
- Python 3.8 или новее
- CUDA (опционально) — если вы используете GPU-ускорение для модели распознавания
- Для локального запуска моделей: свободное место и достаточный объём оперативной памяти
- (Опционально) Graphical environment — для удобного тестирования локального веб-интерфейса

Установка
---------
1. Клонируйте репозиторий:
```bash
git clone https://github.com/Baxa2367/ai_chief.git
cd ai_chief
```

2. Создайте и активируйте виртуальное окружение:
- Linux / macOS:
```bash
python3 -m venv venv
source venv/bin/activate
```
- Windows (PowerShell):
```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

(Если в репозитории нет requirements.txt — рекомендуемые пакеты: `fastapi uvicorn pillow torchvision torch torchvision transformers opencv-python python-multipart aiofiles sqlalchemy` — добавьте по нужде.)

Настройка моделей
-----------------
- Если проект использует предобученные модели (например, для детекции объекта/классификации), укажите путь к модели в файле конфигурации `.env` или `config.py`.
- Для ускоренной работы с GPU — установите PyTorch с поддержкой CUDA согласно вашей версии CUDA.

Пример .env
```
MODEL_PATH=models/ingredient_detector.pt
DB_URL=sqlite:///data/recipes.db
HOST=0.0.0.0
PORT=8000
USE_GPU=false
```

Быстрый старт (локальный веб-интерфейс)
--------------------------------------
1. Запустите приложение (пример для FastAPI + Uvicorn):
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```
2. Откройте браузер: http://localhost:8000
3. Загрузите фото с ингредиентами, дождитесь результата — AI Chief покажет распознанные ингредиенты и предложит рецепты.

REST API (пример)
-----------------
- POST /api/v1/analyze — загрузить фото (multipart/form-data) и получить распознанные ингредиенты + рекомендации рецептов.
- GET /api/v1/recipes — получить список доступных рецептов.
- POST /api/v1/recipes — добавить пользовательский рецепт.
- GET /api/v1/recipes/{id} — получить рецепт по id.

Пример запроса на /api/v1/analyze (curl):
```bash
curl -X POST "http://localhost:8000/api/v1/analyze" \
  -F "file=@/path/to/photo.jpg"
```

Подбор рецептов и база знаний
----------------------------
- Рецепты могут храниться в JSON-файлах или в базе данных.
- Для улучшения подборки используйте дополнительные признаки: сезонность, диета, предпочитаемые кухни, время приготовления.
- Можно внедрять ранжирование результатов с учётом веса совпадения ингредиентов, времени и предпочтений пользователя.

Приватность и безопасность
--------------------------
- В локальном режиме изображения не покидают вашу машину — это ключевой сценарий приватности.
- Если вы добавляете облачные API (например, внешние OCR/vision сервисы), чётко документируйте это и дайте пользователю возможность включать/отключать интеграции.
- Не храните пользовательские данные в открытых репозиториях.

Тестирование
-----------
- Добавьте unit и integration тесты для:
  - корректности распознавания (mock моделей);
  - API-эндпоинтов;
  - маппинга ингредиентов → рецепты.
- Запуск тестов:
```bash
pytest tests/
```

Часто возникающие вопросы (FAQ)
------------------------------
Q: Работает ли проект офлайн?
A: Да — если вы заранее загрузите / используете локальные модели и базы рецептов.

Q: Какие модели используются?
A: Зависит от реализации. Обычно используют модели для детекции/классификации объектов (ResNet, EfficientNet, YOLO-подобные) или мультизаданные vision+NLP (CLIP) для распознавания кухонных ингредиентов.

Q: Нужен ли GPU?
A: Для инференса небольших моделей GPU не обязателен, но ускоряет обработку. Для финетюнинга GPU настоятельно рекомендуется.
